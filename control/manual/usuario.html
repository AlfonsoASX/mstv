Panorama

Qué es MSTV Control

MSTV Control es una plataforma para administrar operación de personal, turnos, checadas, incidencias, comunicación interna, RH, nómina, reportes y permisos. Tiene dos entradas principales:

Panel Control

Usado por administración, RH, supervisores, nómina, clientes y dueños para consultar información y operar módulos internos.

/control/index.php

App móvil

Usada por guardias y colaboradores para ver su turno, checar entrada/salida, reportar incidencias y enviar mensajes.

/app/index.html
Idea clave: el sistema busca que cada registro operativo tenga evidencia: usuario, horario, sitio, geocerca, selfie, estado y bitácora.
Primeros pasos

Acceso, menú y navegación

  1. Entrar al sistema. Abre la URL de Control, captura usuario y contraseña, y presiona iniciar sesión.
  2. Usar el menú lateral. El menú muestra solo las secciones permitidas para tu rol. Si no ves una opción, solicita revisión de permisos.
  3. Buscar en listas. Las tablas principales incluyen buscadores para filtrar por colaborador, No. empleado, sitio, estado o concepto.
  4. Salir correctamente. Usa el botón Salir al final del menú para cerrar la sesión.

Dashboard Operación diaria

El dashboard es la primera lectura del estado general: puntualidad, incidencias recientes, guardias activos, turnos extra, checadas por día y resumen por sitio. Sirve para detectar riesgos antes de entrar al detalle operativo.

Lo usan
  • Dirección
  • Supervisión
  • RH
  • Cliente autorizado
Acciones
  • Revisar KPIs
  • Identificar sitios con baja puntualidad
  • Abrir incidencias o checadas relacionadas
Errores comunes
  • Interpretar porcentajes sin revisar periodo
  • No validar sitios con pocos registros
  • Confundir checadas rechazadas con ausencias

Cómo leerlo correctamente

Úsalo como semáforo inicial, no como cierre definitivo. Si un indicador aparece fuera de rango, entra al módulo relacionado para confirmar detalle: checadas para puntualidad, incidencias para alertas y turnos para cobertura.

  • Primero revisa el periodo o fecha que está mostrando la pantalla.
  • Después compara sitios con mayor actividad contra sitios con alertas.
  • Finalmente abre la lista fuente antes de tomar decisiones de nómina o disciplina.

Turnos Agenda operativa

El módulo de turnos organiza quién debe estar, en qué sitio y a qué hora. Permite ver calendario, capturar turnos individuales, generar horarios recurrentes por semana o mes y registrar turnos extra.

Pasos clave
  • Seleccionar colaborador activo
  • Elegir sitio obligatorio
  • Definir inicio/fin
  • Generar semana o mes
Campos críticos
  • Hora inicio
  • Hora fin
  • Sitio/geocerca
  • Tipo extra u ordinario
Reglas
  • Turno nocturno termina al día siguiente
  • Se evitan duplicados por hora inicio
  • La app depende de turnos reales

Flujo recomendado

Para operación diaria, confirma primero el sitio y el colaborador activo. Si el horario se repite, usa generación semanal o mensual para evitar capturar día por día; después revisa el calendario para detectar huecos o duplicados.

  • Si la hora fin es menor o igual a la hora inicio, trátalo como turno nocturno.
  • No generes turnos sin sitio: la app necesita geocerca para permitir checadas.
  • Antes de nómina, revisa turnos extra y turnos ordinarios del periodo.

Checadas Evidencia

Checadas muestra entrada/salida real contra horario programado, estado de geocerca, selfie, vida, puntaje facial y comentarios. Es el punto de auditoría para aceptar, revisar o explicar registros.

Qué revisar
  • Hora programada
  • Hora real
  • Estado
  • Score facial
Estados
  • ACEPTADO
  • RECHAZADO_GPS
  • RECHAZADO_ROSTRO
  • PENDIENTE_REVISION
Errores comunes
  • Sin foto base
  • Selfie inválida
  • Fuera de geocerca
  • Turno no asignado

Checklist de revisión

Una checada confiable debe tener turno relacionado, hora programada, hora real, ubicación dentro de geocerca y evidencia de selfie. Si una parte falta, no la borres: márcala o consúltala como pendiente para conservar trazabilidad.

  • Compara siempre hora programada contra hora real.
  • Si aparece PENDIENTE_REVISION, revisa foto base, selfie y GPS.
  • Si falta turno, corrige asignación antes de interpretar la asistencia.

Incidencias Atención

Centraliza reportes de seguridad, operación o servicio. Cada incidencia debe revisarse por prioridad, sitio, responsable y estado para cerrar el ciclo de atención.

Acciones
  • Filtrar por prioridad
  • Revisar descripción
  • Dar seguimiento
  • Cerrar cuando proceda
Campos
  • Sitio
  • Reportó
  • Tipo
  • Prioridad
  • Estado
Buenas prácticas
  • No dejar críticas abiertas
  • Registrar notas claras
  • Relacionar con turno si aplica

Ciclo de atención

La incidencia debe pasar de reporte a seguimiento y cierre. Para que sirva como evidencia, la descripción debe ser clara, el sitio correcto y el estado debe actualizarse cuando alguien atienda el caso.

  • Prioriza primero incidentes críticos o relacionados con cliente.
  • Usa notas de seguimiento para evitar perder contexto entre turnos.
  • Revisa reportes de incidencias para detectar patrones repetidos.

Bitácora Auditoría

La bitácora reúne movimientos del sistema, checadas recientes e incidencias recientes. Ayuda a responder quién hizo qué, cuándo ocurrió y qué registros se afectaron.

Uso
  • Auditoría interna
  • Seguimiento de cambios
  • Revisión de operación reciente
Datos
  • Fecha
  • Usuario
  • Acción
  • Tabla/registro
Cuándo verla
  • Antes de corregir datos
  • Después de eliminar movimientos
  • En incidentes operativos

Cuándo consultarla

Consulta bitácora cuando necesites reconstruir una secuencia: cambios de datos, eliminación de movimientos manuales, registros de checada o actividad reciente de usuarios. Es especialmente útil antes de corregir información sensible.

  • Busca por fecha aproximada y usuario involucrado.
  • Compara el evento con la pantalla donde se originó.
  • Conserva bitácora como respaldo operativo, no como sustituto de reportes.

Errores comunes

Uso operativo

El chat sirve para comunicación rápida con colaboradores. No sustituye incidencias formales: si el mensaje describe un problema operativo, conviene levantar o relacionar una incidencia para que quede en seguimiento.

  • Selecciona la conversación antes de responder.
  • Revisa mensajes marcados por moderación antes de tomar acción.
  • Usa mensajes concretos y evita enviar instrucciones ambiguas.

Personal RH

Personal concentra datos generales, laborales, bancarios, emergencia, uniforme, foto base, vacaciones, faltas, descansos, caja, préstamos, descuentos, sanciones, bonos, bajas y ficha técnica.

Pasos clave
  • Alta de colaborador
  • Completar ficha
  • Cargar foto base
  • Revisar No. empleado
Campos críticos
  • Fecha ingreso
  • Estado activo
  • Salario
  • Datos bancarios
  • Contacto emergencia
Reglas
  • No. empleado calculado
  • Vacaciones por historial
  • Caja manual eliminable
  • Nómina generada no se borra manualmente

Orden sugerido de captura

Captura primero identidad, fecha de ingreso y estado laboral; después completa datos de contacto, banco, emergencia, uniforme y foto base. La fecha de ingreso alimenta antigüedad, No. empleado y vacaciones.

  • Usa la ficha para revisar historial completo del colaborador.
  • Registra vacaciones como movimientos, no como notas planas.
  • Revisa datos bancarios y salario antes de calcular nómina.

Sitios y geocercas Ubicación

Administra clientes, sitios, dirección, latitud, longitud y radio de geocerca. Es indispensable para que la app valide dónde se registra cada checada.

Acciones

  • Editar coordenadas
  • Ajustar radio
  • Desactivar sitio
Campos
  • Cliente
  • Nombre
  • Dirección
  • Lat/Lng
  • Radio
Validación
  • Latitud -90 a 90
  • Longitud -180 a 180
  • Radio mayor a 0

Validación de geocerca

Un sitio mal ubicado provoca rechazos o pendientes de checada. Antes de operar, valida que el punto del mapa coincida con el acceso real y que el radio cubra el área permitida sin volverse demasiado amplio.

  • Usa coordenadas precisas, no solo dirección textual.
  • Ajusta el radio si hay entradas válidas fuera por pocos metros.
  • Desactiva sitios que ya no deben recibir turnos.

Nómina Cálculo

Nómina consolida salario, retardos, horas extra, turnos extra, vacaciones, prima vacacional, bonos, faltas, descansos, sanciones, material, Infonavit, Fonacot, préstamos, adelantos, caja y neto.

Antes de calcular
  • Revisar checadas
  • Validar faltas/descansos
  • Confirmar vacaciones
  • Capturar ajustes
Salidas
  • Resumen
  • Conceptos
  • Recibo interno
  • CSV
Errores comunes
  • Periodo incorrecto
  • Datos de salario incompletos
  • Incidencias sin registrar

Secuencia segura

Antes de calcular, asegúrate de que las fuentes operativas ya estén revisadas. Nómina no inventa datos: consolida lo que exista en turnos, checadas, vacaciones, faltas, descansos, caja, préstamos y capturas manuales.

  • Valida el periodo antes de ejecutar cálculo.
  • Revisa conceptos separados de faltas y descansos.
  • Abre el recibo interno para confirmar que el desglose sea explicable.

Cómo usarlos

Administración responsable

Los permisos afectan tanto lo que aparece en menú como el acceso directo por URL. Cambia permisos con cuidado y prueba con un usuario del rol antes de dar por cerrada la configuración.

  • No compartas cuentas entre personas.
  • Usa DUEÑO solo para administración crítica.
  • No retires permisos críticos sin tener una cuenta alternativa de soporte.

Configuración Parámetros

Configuración concentra valores que cambian la operación sin editar código: tolerancia de retardos, anticipación de checadas, score facial, vacaciones, faltas, descansos, turnos extra, préstamos y caja.

Quién la usa
  • Administrador
  • RH
  • Nómina
  • DUEÑO
Ejemplos
  • 20 min anticipación
  • 35 score facial
  • Tabla vacaciones
  • Montos faltas
Cuidado
  • Cambiar valores afecta cálculos futuros
  • Documentar cambios
  • Probar nómina antes de cerrar

Antes de cambiar valores

La configuración modifica reglas de operación. Cambiar tolerancias, vacaciones, facial o descuentos puede afectar checadas y cálculos futuros, por eso conviene documentar el cambio y probar un caso antes de operar masivamente.

  • Anota qué valor cambió, por qué y desde cuándo aplica.
  • Prueba checada o nómina en un caso controlado.
  • Evita dejar campos vacíos si el negocio requiere un valor explícito.

Experiencia del colaborador

La app debe guiar al colaborador a confirmar turno, sitio y horario antes de checar. Si no aparece turno o la ubicación falla, el problema suele venir de asignación, geocerca, permisos del navegador o conectividad.

  • Permitir ubicación y cámara en el navegador.
  • Checar solo dentro de la ventana permitida.
  • Reportar incidencias desde app cuando ocurra algo durante el servicio.
Guardias y colaboradores

Funcionamiento de la app móvil

El colaborador entra con usuario y contraseña. La app valida que exista un registro de personal activo asociado a ese usuario.

La app consulta el turno actual, muestra hora programada y permite registrar entrada desde 20 minutos antes. La salida se habilita después de entrada. Cada registro incluye GPS, selfie y estado.

El colaborador selecciona sitio, tipo, prioridad y descripción. El reporte queda disponible en la bandeja de incidencias para seguimiento.

Permite enviar mensajes a RH/administración. La app refresca conversaciones automáticamente sin recargar toda la pantalla.

Guías paso a paso

Flujos frecuentes

Crear un colaborador
  1. Ir a Personal y seleccionar Nuevo colaborador.
  2. Capturar datos de usuario, rol, nombre, teléfono, fecha de ingreso, estado y datos laborales.
  3. Completar datos personales, bancarios, uniforme y emergencia.
  4. Guardar. El No. empleado se calcula automáticamente con año de ingreso + ID.
  5. Entrar a la ficha para cargar foto base y completar nómina/prestaciones.
Asignar horario semanal o mensual
  1. Ir a Turnos y Asignación.
  2. Seleccionar colaborador activo y sitio obligatorio.
  3. Capturar hora inicio, hora fin, días de la semana y rango semana/mes.
  4. Generar turnos. El sistema evita duplicados por colaborador y hora de inicio.
  5. Verificar que aparezcan en calendario y app móvil.
Registrar una checada desde app
  1. Entrar a la app móvil.
  2. Revisar turno actual, sitio, entrada programada y salida programada.
  3. Esperar a estar dentro de la ventana permitida: 20 minutos antes del inicio.
  4. Permitir ubicación, tomar selfie y registrar entrada o salida.
  5. Confirmar en bitácora que aparezca la hora real.
Capturar vacaciones y consultar historial
  1. Entrar a la ficha del colaborador.
  2. Ubicar el módulo Vacaciones.
  3. Registrar movimiento: gozadas, pagadas, ajuste a favor, ajuste en contra o nota histórica.
  4. Revisar resumen: generadas, gozadas, pagadas, ajustes y disponibles.
  5. Consultar la tabla/timeline para auditoría histórica.
Calcular nómina y emitir recibo
  1. Revisar checadas, retardos, faltas, descansos, bonos, sanciones, préstamos y caja.
  2. Ir a Nómina y seleccionar el periodo.
  3. Capturar ajustes manuales cuando aplique.
  4. Calcular nómina y revisar neto, percepciones y deducciones.
  5. Abrir Recibo para imprimir/guardar PDF o Exportación para CSV.
Administrar permisos por rol
  1. Entrar a Roles y permisos con un perfil autorizado.
  2. Seleccionar el rol a configurar.
  3. Activar secciones y páginas permitidas.
  4. Guardar cambios. DUEÑO conserva acceso total fijo.
  5. Probar con un usuario del rol que el menú y acceso por URL respeten la matriz.
Reglas del negocio

Políticas aplicadas por el sistema

PolíticaCómo funcionaDónde se ve
No. empleadoSe calcula como últimos 2 dígitos del año de ingreso + ID con 4 dígitos. Ejemplo: 260001.Personal, usuarios, nómina, reportes, checadas y ficha.
Ventana de checadaLa entrada se permite desde 20 minutos antes del turno, configurable.App móvil y configuración global.
GeocercaEl sitio define latitud, longitud y radio. La app envía GPS para validar si está dentro.Sitios, checadas y detalle.
Reconocimiento facial básicoSe compara selfie contra foto base. Si falta foto base o no hay score calculable, queda pendiente de revisión.App móvil, checadas y ficha.

Casos de uso

Historias de usuario por perfil

Como guardia

Quiero ver mi turno, checar entrada/salida con ubicación y selfie, reportar incidencias y comunicarme con administración.

Como supervisor

Quiero revisar checadas, incidencias y turnos para actuar rápidamente cuando falte personal o exista una alerta.

Como RH

Quiero mantener completa la ficha del colaborador, registrar vacaciones, faltas, descansos, préstamos, caja y bajas.

Como nómina

Quiero calcular periodos con incidencias separadas, exportar datos y emitir recibos internos claros.

Como administrador

Quiero configurar permisos, variables globales y usuarios para controlar el acceso al sistema.

Como cliente

Quiero consultar métricas y reportes autorizados sin ver información administrativa sensible.

Uso correcto

Buenas prácticas operativas

Antes de nómina: revisa checadas pendientes, faltas, descansos, vacaciones, bonos, sanciones, préstamos, caja y turnos extra.
Antes de editar permisos: confirma que no estés quitando acceso crítico a RH, nómina o administración.
Antes de generar turnos masivos: valida sitio, días de semana, hora inicio y hora fin.
Con checadas pendientes: revisa selfie, GPS, score facial y comentarios antes de aceptar o rechazar operativamente.
